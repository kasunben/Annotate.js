# Annotate.js

Lightweight vanilla JS library that adds inline annotation and threaded comments to any web page via a single `<script>` tag. Zero runtime dependencies.

> **Git workflow:** Do NOT commit or push to the remote repository unless the user explicitly asks. Make changes locally and wait for instruction before staging, committing, or pushing.

## Integration

```html
<!-- Development / local (raw source, no Trystero bundle) -->
<script src="annotate.js" data-site-id="your-site-id"></script>

<!-- CDN ‚Äî offline / P2P (no server needed; annotate.min.js committed to repo, served by jsDelivr) -->
<script src="https://cdn.jsdelivr.net/gh/kasunben/Annotate.js@latest/annotate.min.js"
        data-site-id="your-site-id"></script>

<!-- CDN ‚Äî pinned version (recommended for production) -->
<script src="https://cdn.jsdelivr.net/gh/kasunben/Annotate.js@v0.3.5/annotate.min.js"
        data-site-id="your-site-id"></script>

<!-- Production ‚Äî server sync (self-hosted) -->
<script src="https://your-server.example.com/annotate.min.js"
        data-site-id="your-site-id"
        data-sync-url="https://your-server.example.com"></script>

<!-- Production ‚Äî P2P mode (no server needed; use a UUID as room-id) -->
<script src="https://cdn.jsdelivr.net/gh/kasunben/Annotate.js@latest/annotate.min.js"
        data-site-id="your-site-id"
        data-room-id="f3a9c271-8d4e-4b1a-9c3f-d17b2e5a08cc"></script>

<!-- P2P with self-hosted relay (enterprise) -->
<script src="https://cdn.jsdelivr.net/gh/kasunben/Annotate.js@latest/annotate.min.js"
        data-site-id="your-site-id"
        data-room-id="f3a9c271-8d4e-4b1a-9c3f-d17b2e5a08cc"
        data-relay-url="wss://your-relay.example.com"></script>
```

| Attribute | Purpose | Default |
|---|---|---|
| `data-site-id` | IDB namespace + server/P2P scope | `"default"` |
| `data-sync-url` | Activates server sync mode | absent = offline |
| `data-room-id` | Activates P2P mode (UUID recommended) | absent = no P2P |
| `data-relay-url` | Override hosted relay for P2P signaling | project's hosted relay |
| `data-admin-id` | UUID of the designated admin browser; that browser's Settings ‚Üí Data section is shown in all sync modes | absent = first-annotator heuristic |
| `data-sync-ms` | Server sync poll interval in milliseconds (Mode 3 only); invalid values fall back to 30 000 with a console warning | `30000` |

`data-sync-url` and `data-room-id` are mutually exclusive ‚Äî the library warns and suppresses P2P if both are set.

Everything (IndexedDB layer, anchor serialization, BroadcastChannel, P2P sync, server sync, UI, CSS) is inlined into the IIFE.
Injects a collapsible right sidebar + floating comment button. All CSS embedded via `document.createElement('style')`.

`annotate.min.js` is committed to the repo on every release (not gitignored). esbuild bundles
Trystero (NOSTR P2P signaling) into the output at build time; no runtime CDN fetch needed.
This means jsDelivr can serve it directly from any git tag:
`https://cdn.jsdelivr.net/gh/kasunben/Annotate.js@<tag>/annotate.min.js`.
The Dockerfile also rebuilds it automatically in stage 1 from source.

## Sync Modes

Four modes, always layered in this order. Modes 1+2 are always active; 3 and 4 are opt-in and mutually exclusive.

| # | Mode | Activated by | Scope | Dependencies |
|---|---|---|---|---|
| 1 | **Offline** | *(default)* | Single tab, IDB-persisted | None |
| 2 | **BroadcastChannel** | *(automatic with Mode 1)* | Same browser, all tabs on same origin, instant | Browser BroadcastChannel API |
| 3 | **Server sync** | `data-sync-url` | Any browser/device, SQLite-durable | Node.js server |
| 4 | **P2P** | `data-room-id` | Any browser, DTLS-encrypted WebRTC | Room UUID (shared secret); relay for signaling only |

### Mode 2 ‚Äî BroadcastChannel
Zero configuration. `_bc = new BroadcastChannel('annotate-' + _siteId)` created at IIFE init. Every `syncThread` / `syncActivity` call posts `THREAD_UPDATE` / `ACTIVITY_UPDATE` before the `if (!_syncUrl)` guard so broadcasts fire in all modes. Cross-browser (e.g. Firefox ‚Üí Brave) requires Mode 4 ‚Äî BroadcastChannel is same-browser-process only by spec.

### Mode 3 ‚Äî Server sync
Push on every mutation (`POST /threads`, `POST /activity`). Pull on page load + 30 s interval + `visibilitychange`. `dirty` flag queues offline writes for `flushDirtyThreads()` on reconnect. Last-write-wins by `updatedAt`; `?since=` timestamps drive incremental pulls.

### Mode 4 ‚Äî P2P (three-tier signaling)
`initP2P()` called in boot sequence when `_roomId` is set. Tries Tier 1 first; 5 s fallback timer fires `_initNostrP2P()` if relay doesn't connect.

| Tier | Function | Method | Fallback trigger |
|---|---|---|---|
| 1 | `_initRelayP2P(relayUrl, ‚Ä¶)` | Hosted Cloudflare Worker WebSocket (`wss://relay.annotate-js.workers.dev`) | `ws.onerror` or 5 s timeout |
| 2 | `_initRelayP2P(relayUrl, ‚Ä¶)` *(same function, different URL)* | Self-hosted relay via `data-relay-url` | Same as Tier 1 |
| 3 | `_initNostrP2P()` | NOSTR public relays via Trystero | None ‚Äî terminal fallback |

Once a WebRTC data channel opens, `_p2pBroadcastThread` / `_p2pBroadcastActivity` / `_p2pBroadcastAdmin` no-op stubs are replaced with real senders that iterate all open peer connections. Annotation data never passes through the relay ‚Äî only SDP offers/answers and ICE candidates do.

### Expected P2P console output (before relay is deployed)
```
WebSocket connection to 'wss://relay.annotate-js.workers.dev/‚Ä¶' failed  ‚Üê expected
Annotate.js P2P: relay disconnected ‚Äî falling back to NOSTR              ‚Üê expected
Trystero: relay failure from wss://relay.damus.io/ ‚Äî rate-limited ‚Ä¶     ‚Üê expected during heavy testing
```
All harmless. P2P works via Tier 3. Disappears once the Cloudflare relay is deployed.

## Data Model

### Thread
```js
{
  id:          string,   // UUID v4, client-generated
  siteId:      string,   // from data-site-id
  pageUrl:     string,   // window.location.href (no hash)

  quote:       string,   // snapshot of selected text
  anchor: {
    xpath:       string, // XPath to text node
    startOffset: number,
    endOffset:   number,
  },

  body:        string,
  author:      string,
  authorId:    string | null,  // UUID from localStorage 'annotate_author_id'; null = legacy (pre-access-control)
  createdAt:   string,   // ISO 8601
  updatedAt:   string,

  resolved:    boolean,
  resolvedAt:  string | null,
  resolvedBy:  string | null,

  replies: [{
    id:        string,
    body:      string,
    author:    string,
    authorId:  string | null,  // same as thread.authorId ‚Äî null for legacy replies
    createdAt: string,
    updatedAt: string | null,
    deleted:   boolean,        // soft-delete
  }],

  dirty:       boolean,        // true = not yet synced to server (client-only field)
  deletedAt:   string | null,  // soft-delete
}
```

### ActivityEntry
```js
{
  id:        string,
  siteId:    string,
  pageUrl:   string,
  type:      'thread_created' | 'thread_resolved'   | 'thread_unresolved'
           | 'thread_deleted' | 'thread_edited'     | 'reply_added'
           | 'reply_edited'   | 'reply_deleted'
           | 'data_exported'  | 'data_imported',
  threadId:  string | null,  // null for data_exported / data_imported (page-level events)
  replyId:   string | null,
  actor:     string,
  timestamp: string,   // ISO 8601
  snapshot:  string,   // short human summary e.g. "replied: 'great point'"
}
```

ActivityEntries are synced to the server when `data-sync-url` is set ‚Äî all users on a page share the same activity log. Each client pushes on write (`POST /activity`) and pulls on page load, 30s poll, and tab focus (`GET /activity?since=`). When `data-sync-url` is absent, activity is IDB-only (original behaviour preserved).

### UserConfig (localStorage only)
```js
{ displayName: string, configuredAt: string }
```

## Storage Architecture

| Data | Store | Why |
|---|---|---|
| Threads | **IndexedDB** (primary) + **SQLite** (server) | IDB is the offline working store; server is canonical for multi-user |
| Activity log | **IndexedDB** (primary) + **SQLite** (server) | Same push/pull pattern as threads; cross-user shared history |
| User config | **localStorage** | Tiny, simple, global |

**IDB name:** `annotate-{siteId}` ¬∑ **Version:** 1

| Object store | Key | Indexes |
|---|---|---|
| `threads` | `id` | `[pageUrl]`, `createdAt` |
| `activity` | `id` | `[pageUrl]`, `timestamp` |

**SQLite** (`server/data/annotate.db`): `threads` table and `activity` table. `anchor` and `replies` stored as JSON columns in `threads`. `dirty` field is client-only and never written to SQLite. Activity entries use `INSERT OR IGNORE` ‚Äî they are immutable once written.

## Highlight Re-anchoring

`Range` objects are ephemeral. On save, serialize to a stable `Anchor`:
```js
// Serialize
anchor = { xpath: getXPath(range.startContainer), startOffset, endOffset }

// Restore on page load
range = document.evaluate(anchor.xpath) ‚Üí set offsets ‚Üí reconstruct Range ‚Üí apply <mark>
```
If XPath no longer resolves (DOM changed), show Thread in sidebar with a
"highlight unavailable" badge rather than dropping it silently.

## Sync Architecture

Sync is **opt-in** via `data-sync-url`. When absent, the library behaves exactly as before (IDB-only).

### Strategy

| Trigger | Action |
|---|---|
| Every local thread mutation | `syncThread(t)` ‚Äî fire-and-forget `POST /threads` (upsert) |
| Every local activity event | `syncActivity(entry)` ‚Äî fire-and-forget `POST /activity` |
| Page load | `pullThreads()` + `pullActivity()` ‚Äî full pull; `flushDirtyThreads()` ‚Äî push offline mutations |
| Every 30 seconds | `pullThreads()` + `pullActivity()` ‚Äî incremental pull via `?since=` |
| Tab becomes visible | `pullThreads()` + `pullActivity()` ‚Äî incremental pull |

### Conflict resolution

**Last-write-wins by `updatedAt`.** Server wins for non-dirty local records on pull. Local dirty records are skipped during pull and pushed on the next `syncThread` or `flushDirtyThreads` call.

`_lastSync` is set to the max `updatedAt` seen in each server response (not client clock), avoiding clock skew issues.

### `_rerenderAfterPull(serverThreads)`

Delta DOM update after every pull ‚Äî no full page reload:
- **Deleted** thread: remove card from sidebar + unwrap `<mark>`
- **Resolved** thread (from another user): dim active card or apply `.is-resolved` mark
- **New** active thread: `restoreRange` + `renderThreadCard`
- **Updated** active thread: `_renderSavedCard` ‚Äî skipped if a composer textarea is open (open-composer guard prevents wiping a draft)

## REST API

| Method | Path | Body | Action |
|---|---|---|---|
| GET | `/threads` | `?siteId&pageUrl[&since]` | All threads for page; `since` enables incremental pull |
| POST | `/threads` | full Thread object | Upsert (`INSERT OR REPLACE`) |
| PATCH | `/threads/:id` | `{ body, updatedAt }` | Edit body |
| PATCH | `/threads/:id/resolve` | `{ resolvedBy, resolvedAt }` | Resolve |
| DELETE | `/threads/:id` | `{ deletedAt }` | Soft-delete |
| POST | `/threads/:id/replies` | `{ reply }` | Append reply |
| PATCH | `/threads/:id/replies/:replyId` | `{ body, updatedAt }` | Edit reply |
| DELETE | `/threads/:id/replies/:replyId` | ‚Äî | Soft-delete reply (`deleted: true`) |
| DELETE | `/threads` | `?siteId=` | Hard-delete all threads for a site (Settings ‚Üí Clear all) |
| GET | `/activity` | `?siteId&pageUrl[&since]` | All activity for page; `since` enables incremental pull |
| POST | `/activity` | full ActivityEntry object | `INSERT OR IGNORE` ‚Äî entries are immutable |
| DELETE | `/activity` | `?siteId=` | Hard-delete all activity for a site (Settings ‚Üí Clear all) |

All thread mutation endpoints return the full updated Thread object so the client can `dbSaveThread` directly.
GET /threads includes soft-deleted threads so deletes propagate to other clients on pull.
Activity entries are never updated ‚Äî `INSERT OR IGNORE` ensures idempotent pushes.

## Implementation Phases

### Phase A ‚Äî Persistence ‚úÖ
1. ‚úÖ IndexedDB layer ‚Äî inlined into `annotate.js`
2. ‚úÖ Anchor layer ‚Äî `serializeRange` / `restoreRange`
3. ‚úÖ Wire mutations ‚Üí IDB, load + re-render on page load
4. ‚úÖ Display name prompt on first annotation

### Phase B ‚Äî Tab views ‚úÖ
- ‚úÖ Resolved tab, Activity tab, Settings tab

### Phase C ‚Äî Backend sync ‚úÖ
- ‚úÖ Node.js + Express + `node:sqlite` REST server (`server/`)
- ‚úÖ Sync layer inlined into `annotate.js`: `syncThread`, `pullThreads`, `flushDirtyThreads`, `_rerenderAfterPull`
- ‚úÖ Multi-user: 30s poll + visibility-change refresh

### Phase D ‚Äî Build & Deploy ‚úÖ
- ‚úÖ `npm run build` ‚Äî esbuild bundles + minifies `assets/js/annotate.js` ‚Üí `annotate.min.js` (project root, served at `/annotate.min.js`)
- ‚úÖ `Dockerfile` ‚Äî multi-stage build; lean Node 23 Alpine runtime image
- ‚úÖ `docker-compose.yml` ‚Äî pre-built GHCR image + named SQLite volume + PORT override
- ‚úÖ `ecosystem.config.js` ‚Äî PM2 config; `instances: 1` enforced for SQLite write-lock safety
- ‚úÖ `.github/workflows/docker-publish.yml` ‚Äî publishes `linux/amd64` + `linux/arm64` image to `ghcr.io/kasunben/annotate.js` on `v*.*.*` git tags

### Phase E ‚Äî Server-side activity log ‚úÖ
- ‚úÖ `activity` table in SQLite ‚Äî `INSERT OR IGNORE`; immutable entries
- ‚úÖ `GET/POST/DELETE /activity` endpoints in `server/routes/activity.js`
- ‚úÖ `syncActivity()` + `pullActivity()` in `annotate.js` ‚Äî same push/pull pattern as threads
- ‚úÖ `logActivity()` helper ‚Äî replaces all 7 `dbAddActivity(makeActivity(...))` call sites
- ‚úÖ Clear all ‚Äî wipes server activity alongside threads via `Promise.all`

### Phase P ‚Äî P2P Sync Mode ‚úÖ
Three-tier signaling: hosted Cloudflare relay (default) ‚Üí self-hosted relay (`data-relay-url`) ‚Üí NOSTR fallback (auto).
Annotation content flows peer-to-peer over DTLS-encrypted WebRTC data channels ‚Äî no relay sees it.
See `docs/rfc-p2p-sync.md` for full architecture decisions.

- ‚úÖ **P1 ‚Äî BroadcastChannel** ‚Äî zero-dep same-origin multi-tab sync; wired into `syncThread` + `syncActivity`
- ‚úÖ **P2 ‚Äî Cloudflare Worker relay** ‚Äî `relay/worker.js` + `relay/wrangler.toml`; ephemeral Durable Object room state
- ‚úÖ **P3 ‚Äî WebRTC P2P in `annotate.js`** ‚Äî `data-room-id` activates; `initP2P()` ‚Üí relay tier ‚Üí NOSTR fallback; esbuild bundles Trystero

### Phase G ‚Äî Ownership-based access control ‚úÖ
- ‚úÖ `_authorId` UUID per browser (`localStorage` key `annotate_author_id`); attached to every Thread + Reply
- ‚úÖ `_isOwner(item)` ‚Äî unrestricted in offline/BC mode; ownership check in server-sync / P2P
- ‚úÖ `_renderSavedCard` / `_buildReplyEl` ‚Äî conditional Edit/Delete menu rendering
- ‚úÖ Server `checkOwnership` on `POST /threads` upsert ‚Äî 403 on mismatch; `author_id` column added to SQLite with idempotent `ALTER TABLE` migration
- ‚úÖ `DELETE /threads?authorId=` scopes deletion; client uses `dbClearMyThreads` in server-sync; settings clear button hidden in server-sync + P2P
- ‚úÖ `POST /threads` validates `id` / `siteId` / `pageUrl` and returns 400 JSON on missing body (no more 500 HTML crash page on malformed input)

### Phase H ‚Äî UI polish ‚úÖ

### Phase I ‚Äî Export / Import ‚úÖ
- ‚úÖ **`_exportData()`** ‚Äî downloads all threads (active + resolved + soft-deleted), activity entries, and settings (displayName + authorId) for the current `_pageUrl` as a JSON file; filename: `annotate-{siteId}-{YYYY-MM-DD}.json`; uses `_dbGetByPage` (returns all threads including deleted) + `dbGetActivity`; logs `data_exported` activity entry after download

- ‚úÖ **`_adminId`** ‚Äî read from `document.currentScript.dataset.adminId`; `null` if absent

## Key Implementation Notes

- **IIFE pattern** ‚Äî all client code scoped in `(function() { ... })()` to avoid globals; esbuild wraps this in an outer IIFE via `--format=iife`, making it a nested IIFE (harmless)
- **`siteId`** ‚Äî read via `document.currentScript.dataset.siteId`; IDB namespace + server/P2P scope
- **`syncUrl`** ‚Äî read via `document.currentScript.dataset.syncUrl`; `null` = sync disabled
- **`roomId`** ‚Äî read via `document.currentScript.dataset.roomId`; `null` = P2P disabled; activates `initP2P()` in boot sequence only when `_syncUrl` is absent (mutual exclusivity enforced)
- **`relayUrl`** ‚Äî read via `document.currentScript.dataset.relayUrl`; defaults to `'wss://relay.annotate-js.workers.dev'`; overridden by `data-relay-url` for self-hosted relay
- **`_pollMs`** ‚Äî read from `data-sync-ms` via `document.currentScript.dataset.syncMs`; validated via `parseInt > 0`; falls back to `30000` with a `console.warn` if the attribute is present but invalid (non-numeric, zero, or negative); replaces the hardcoded `30000` in the `setInterval` for server sync; absent attribute is the same as `30000` (no warning)
- **Mutual exclusivity guard** ‚Äî if both `_syncUrl` and `_roomId` are set, `console.warn` fires at IIFE init and `initP2P()` is not called (`if (_roomId && !_syncUrl)`); server sync takes precedence
- **BroadcastChannel** ‚Äî `_bc` created for same-origin multi-tab sync; posts `THREAD_UPDATE` / `ACTIVITY_UPDATE` messages; handler in `_bc.onmessage` applies last-write-wins before re-rendering; gracefully absent (`null`) in environments without BroadcastChannel support
- **BroadcastChannel `>=` not `>` for updatedAt** ‚Äî IDB is shared across all same-origin tabs in the same browser; when Tab 1 calls `dbSaveThread` and then (inside `.then()`) posts the BC message, the write has already committed to shared IDB by the time Tab 2's handler calls `dbGetThread`; so `existing.updatedAt === msg.thread.updatedAt` for brand-new threads, and a strict `>` comparison would skip the re-render entirely; the condition must use `>=` so equal timestamps still trigger `_rerenderAfterPull`
- **P2P tiered signaling** ‚Äî `initP2P()` tries the relay first with a 5 s fallback timer; `_initRelayP2P(url, onConnected, onFailure)` calls `onConnected()` on `ws.onopen` (clears timer); `onFailure()` fires on `ws.onerror`; both paths lead to `_initNostrP2P()` as fallback
- **`_p2pBroadcastThread` / `_p2pBroadcastActivity`** ‚Äî no-op function stubs replaced by real senders once a P2P connection is established; called at the top of `syncThread` / `syncActivity` so broadcasts fire in offline-only mode too
- **Trystero bundled at build time** ‚Äî raw `annotate.js` has **no `import` statement** (it must remain loadable as a plain classic `<script>`); instead, `assets/js/trystero-shim.js` exports `{ joinRoom as _trysteroJoin } from 'trystero/nostr'` and the build command uses `--inject:assets/js/trystero-shim.js`; esbuild replaces every free-variable `_trysteroJoin` reference with the bundled function at build time; when the raw source is loaded directly, `_trysteroJoin` is `undefined` and the null guard in `_initNostrP2P` (`if (typeof _trysteroJoin !== 'function') return;`) short-circuits cleanly; final bundle ~64 KB minified
